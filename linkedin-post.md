**Which path are you navigating in the AI revolution?**

Open Source Builder, for me. Latest: luxuriously usable flyout menus.

Fisheye Menu is a Fitts's Law optimized cascading menu where items near your cursor grow larger — bigger targets = faster selection. Built in one evening with Claude Code, from concept to published npm-ready library with 186 tests, keyboard navigation, ARIA accessibility, and a proper academic bibliography.

This is a recreation of work from my MS in Human Factors at Clemson (~2005). Back then it was a prototype in a research lab. Twenty years later, AI did something I didn't expect: it searched the literature, found Cockburn's 2007 paper that dismissed fisheye menus as slower — then recognized that my approach (fixed total height, proportional redistribution) avoids the exact problem Cockburn identified. It wrote the steering corridor trig. It generated the test suite that caught boundary violations I wouldn't have found manually.

The code is the easy part. The hard part was always knowing *what* to build and *why* it matters. That part's still human.
